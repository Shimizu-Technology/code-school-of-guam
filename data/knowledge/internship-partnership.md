# Internship Partnership Overview

## Code School of Guam × Shimizu Technology LLC

## 1. Internship Purpose

Provide graduating CSG students with hands-on, résumé-ready experience while delivering real business value to Shimizu Technology's client and internal projects.

## 2. Host Company Snapshot

**Shimizu Technology** – Guam-based software development company

### Tech Stack
- **Backend**: Ruby on Rails, Python/FastAPI
- **Frontend**: React.js, React Native, Next.js
- **AI**: OpenAI SDK, Pinecone (RAG), LangChain

### Current Projects
- **HafaGPT** – AI-powered Chamorro language learning platform (React + FastAPI)
- **HafaRecipes** – AI-powered recipe management app (React Native + FastAPI)
- **Airport Golf System** – Tournament registration and management (React + Rails)
- **Hafaloha** – Restaurant ordering and POS system (React + Rails)
- **Order Suite** – Online ordering platform for restaurants
- Additional client projects as needed

## 3. Internship Structure

- **Length**: 10 weeks – five two-week Agile sprints
- **Sprint 1 (Weeks 1-2)**: Onboarding – clone repos, run projects locally, read docs, pair with mentor to learn architecture
- **Sprints 2-5 (Weeks 3-10)**: Each intern owns 2-3 scoped tickets per sprint (features, bugs, refactors)
- **Environment**: 100% remote (Zoom, Slack, GitHub)
- **Stand-ups / Check-ins**: Tuesday and Thursday meetings (30 min each)

### Ceremonies
- **Sprint Planning** – First Tuesday of each sprint
- **Sprint Review + Retro** – Thursday at sprint end

## 4. Roles & Responsibilities

### Shimizu Technology
- Provide backlog, onboarding docs, and mentorship
- Facilitate sprint ceremonies and code reviews
- Assign appropriate tasks based on skill level

### Code School of Guam
- Match interns to projects based on skills and interests
- Monitor progress and provide support
- Coordinate with Shimizu Technology on any issues

### Interns
- Attend all scheduled meetings
- Deliver assigned tickets by sprint's end
- Follow team Git workflow (branches, PRs, code reviews)
- Present mid-term and final demos
- Communicate proactively about blockers or questions

## 8. Supervision & Evaluation

- Weekly check-ins with Sprint Lead
- Mid-point review (Week 5) – progress demo + feedback
- Final demo (Week 10) – walk-through of completed work + reflection

### Evaluation Rubric Covers
- Technical quality
- Communication
- Teamwork
- Professionalism
- Growth and initiative

## 9. Tools & Workflow

### Project Management
- **ClickUp** – Kanban boards for task tracking
- Columns: To Do → In Progress → In Review → Done
- Labels: Priority (Easy/Medium/Hard) and Project tags

### Git Workflow
1. Create branch from main: `feature/task-name` or `fix/bug-name`
2. Make changes, commit often with clear messages
3. Push branch, create Pull Request
4. Request review from Sprint Lead or assigned reviewer
5. Address feedback, merge when approved

### Communication
- **Slack** – Daily communication, questions, updates
- **Zoom** – Scheduled meetings
- **GitHub** – Code reviews, PRs, technical discussions

## 10. Logistics & Security

- Private GitHub repos with least-privilege access
- Staging/production environments use encrypted environment variables
- Actions and deployments are logged
- Interns work on staging environments, not production (until approved)
